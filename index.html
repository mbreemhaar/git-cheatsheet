Introduction

Git is a powerful version control system that can be overwhelming for beginners. A Git cheat sheet is a valuable tool to quickly access the most commonly used Git commands and options. This cheat sheet covers essential Git commands to help you become more productive in your development workflow.

Setting up a new repository

The easiest way to set up a new repository is to run git init in the directory where you want your project files to be. The git init command is especially useful when you already have an existing project that is not in a git repository yet.

Another way to set up your git repository, is making a new repository on GitHub. GitHub will provide you with a repository URL that you can use to clone the repository to your computer. You can do this by simply running git clone your-repository-url in the directory where you want your repository to live.

Committing changes

After making some changes to your codebase, you can commit them to the repository. To do this, you will first have to stage your changes. You can do this with git add path/to/file. Here, you can replace path/to/file with the file path to your changed files. You can also use git add . to add all changed files in your current working directory.

If there is anything staged that you do not want to commit just yet, you can use git reset HEAD -- path/to/file to remove a file from the staging area.

When you are happy with the staged files, you can use git commit -m "Your description here" with a short description of your changes to commit them to your repository.

Working with branches

The possibility of using branches is one of the most powerful features of git. You can use branches to work on a part of the code while others are working on other parts of the same codebase. By working on different branches, your changes will not affect your colleagues until the code is merged to a main or master branch.

You can make a new branch with git branch branch-name. After doing this, you can move to the new branch using git checkout branch-name. To see a list of all available branches in your local repository, simply use git branch.

When you want to merge the changes of your branch to another branch, you first have to go to the branch you want to merge your changes to with git checkout branch-name. After doing this, you can merge the changes of your branch with git merge branch-name.

Using a remote repository

Remote repositories are another powerful feature of git. Using remote repositories, you can synchronize your repository with one on a git server, for example GitHub.

If you used git clone to initialize your repository, you already have your remote branch set up. Otherwise, you can connect your local repository to a remote one with git remote add origin your-repository-url. This will add a remote repository with the name origin, which is the standard name for a remote repository.

After connecting a remote repository, you can pull any changes from the remote repository using git pull. Equally, you can push your own changes using git push.

Additional useful commands

  • Show a log of commits: git log
  • Show the current branch and changed files: git status
  • Remove all changes since last commit: git reset --hard
  • Restore to last commit, but save changes: git stash
  • Restore changes that were saved with git stash: git stash pop
